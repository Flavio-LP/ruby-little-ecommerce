# Story 5.3: Seller Order Visibility

## Status
Ready

## Story
**As a** seller,
**I want** to see orders placed in my shop,
**so that** I know what to fulfill.

## Acceptance Criteria
1. `/:shop_slug/admin/orders` lists orders for the current shop only (tenant-scoped via `acts_as_tenant`), ordered most-recent-first.
2. Each order row links to a detail view showing line items, quantities, and total.
3. A request spec verifies a seller cannot view another shop's orders by guessing an order ID (403/404).

## 🤖 CodeRabbit Integration

## Tasks / Subtasks

- [ ] Task 3: Request spec for cross-tenant denial (AC: 3)
- [ ] Task 4: Dashboard "Orders" placeholder card (from Story 2.3) updated to link here

## Dev Notes
- Depends on Story 5.1/5.2 (`Order` model + at least one order created via checkout).
- Same dual-layer pattern as Story 3.2 (`Product`): `acts_as_tenant` default_scope already returns 404 for cross-tenant `Order.find`, `authorize!` is the second layer.
- Routes: `only: %i[index show]`, no create/update/destroy exposed here (orders are created only via checkout, never directly by sellers).
