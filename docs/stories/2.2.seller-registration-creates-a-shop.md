# Story 2.2: Seller Registration Creates a Shop

## Status
Done

## Story
**As a** prospective seller,
**I want** to register an account and have a shop automatically created for me,
**so that** I can start managing my own store immediately after signing up.

## Acceptance Criteria
1. A `Shops::Register` Interactor encapsulates: create `User` with `role: seller`, create associated `Shop` with a slug derived from the shop name (uniqueness enforced, suffixing if needed), wrapped in a database transaction.
2. The seller registration form collects shop name in addition to standard Devise fields.
3. On success, the seller is signed in and redirected to their shop's admin dashboard (`/:shop_slug/admin`).
4. A feature spec (Capybara) covers the full registration-to-dashboard-redirect flow.
5. A failure in shop creation rolls back the user creation (no orphaned `User` without a `Shop`).

## Dev Notes
- Depends on Story 2.1 (`Shop` model, tenant resolution) and Story 2.3 (admin dashboard route must exist for the redirect target — sequence accordingly: implement 2.1 → 2.3's route shell → 2.2, OR implement 2.2's redirect target as a stub here and let 2.3 flesh it out. Recommended: implement Story 2.3's route+controller shell as part of this story too, since AC3 requires the redirect target to exist, then Story 2.3 focuses on the dashboard *content*/cross-tenant denial).
- Interactor pattern (`docs/architecture/components.md`): `.call(context)`, transaction wraps both `User.create!` and `Shop.create!`.
- Slug generation: parameterize the shop name (`shop_name.parameterize`), append a numeric suffix on collision (`-2`, `-3`, ...) until unique.
- This story overrides Devise's default registration controller — only for `role: seller` signups. Customer registration (Epic 4/5, if a dedicated flow is needed) is out of scope here; for now any signup without a shop name defaults to `role: customer` per the existing Devise default flow.
