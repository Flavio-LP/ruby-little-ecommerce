# Story 2.1: Shop Model and Tenant Scoping Infrastructure

## Status
Done

## Story
**As a** developer,
**I want** a `Shop` model and the `acts_as_tenant` configuration wired into `ApplicationController`,
**so that** every tenant-scoped model created in later epics automatically inherits data isolation.

## Acceptance Criteria
1. `Shop` model exists with `name`, `slug` (unique, indexed, URL-safe format validated).
2. `acts_as_tenant` is configured with `Shop` as the tenant class.
3. `ApplicationController` resolves the current tenant from `params[:shop_slug]` via `set_current_tenant_through_filter`, raising a 404 (not a 500) when the slug doesn't match any `Shop`.
4. A request spec verifies that visiting a route with an invalid `shop_slug` returns 404.

## Dev Notes
- Schema detail: `docs/architecture/schema-design.md` — section "Shop" and "acts_as_tenant configuração global". Exact config: `ActsAsTenant.configure { |c| c.require_tenant = true }`.
- `ApplicationController#set_current_shop`:
  ```ruby
  set_current_tenant_through_filter
  before_action :set_current_shop

  private
  def set_current_shop
    shop = Shop.find_by!(slug: params[:shop_slug])
    set_current_tenant(shop)
  rescue ActiveRecord::RecordNotFound
    raise ActionController::RoutingError, "Shop not found"
  end
  ```
  Only applies to controllers that actually have a `:shop_slug` route param — this `before_action` must be skippable/conditional for any future shop-agnostic route (none in Epic 1/2 yet).
- `User#shop_id` column already exists since Story 1.3 (nullable, no FK yet) — this story adds `add_foreign_key :users, :shops` now that `shops` exists, and uncomments `belongs_to :shop` in `app/models/user.rb`.
- Critical rule (`docs/framework/coding-standards.md`): tenant scoping via `acts_as_tenant`, never manual `shop_id` filters.

### Testing
- `spec/models/shop_spec.rb`, `spec/requests/` for the 404 case (needs a throwaway or real tenant-scoped route — Story 2.3 introduces the first real one; this story can test against a minimal route added here, e.g. a placeholder `/:shop_slug/ping`, OR defer the request spec until Story 2.3's admin route exists. Recommended: add a minimal `GET /:shop_slug` route in this story rendering 204, solely to prove tenant resolution, removed/superseded once Story 2.3 adds the real dashboard route — avoid scope creep into building UI here).
